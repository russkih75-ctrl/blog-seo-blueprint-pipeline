# MASTER PROMPT — автоматизация «Вордпресс статьи» (100% чеклист)

Используй этот документ как **единый регламент**: выполняй шаги по порядку, не пропуская обязательные пункты. Целевой сайт публикации: **https://wordprais.ru/** (политика ссылок и ориентиры — `config/wordpress-articles.json`).  
Статья-образец **только по структуре блоков** (mayai и др.): **не копировать** текст, факты, лица и промо чужих авторов.

**Компактная шпаргалка для агента (меньше токенов):** `MASTER_PROMPT_SHORT.md` — держать те же обязательства, полный файл — для людей и аудита.

---

## Фаза A — Постановка задачи (Intake)

1. Зафиксируй **тему**, **нишу**, **ключевые фразы**, **интент** (инфо / коммерция / смешанный), язык **русский**.
2. Запроси или возьми из ТЗ **URL статьи-образца структуры** (например материал с нужным каркасом секций). Обозначь: *structure-only*.
3. Если нужна обложка/баннер с **конкретным лицом** — получи отдельный **visual reference** (URL изображения). Включи **`identity_lock: true`** в handoff.
4. Цель WordPress: тип записи **`post`**, формат контента **HTML** как на сайте (см. фазу G).

Выход: заполненный блок `intake` в `artifacts/content-runs/<runId>/handoff.json`.

---

## Фаза B — Ключи и семантика (аналог модуля «Первый подбор ключей»)

1. Сформируй **ровно три** коротких сид-запроса `k1, k2, k3` под тему (как в blueprint JSON).
2. Зафиксируй в `keywords.json` первичную и вторичные формулировки, сущности, вопросы для GEO.

---

## Фаза C — Research

1. Собери фактуру **без выдуманных процентов и «исследований»** без источника.
2. Прочитай **ориентиры с wordprais.ru** из `config/wordpress-articles.json` → `publicProfileUrls` (главная, блог, услуги WordPress, Яндекс/Google) чтобы **терминология и тон** совпадали со студией.
3. Проверь **пересечение с уже опубликованными темами**: используй индекс антидубля репозитория + при наличии MCP при необходимости **`wordpress_search_posts`** по ключевым фразам (не создавать вторую статью с тем же тезисом).

Выход: `research.md` с пометками источников или «требует верификации».

---

## Фаза D — SEO / GEO каркас

1. Подготовь **SEO title**, **meta description**, **slug** (латиница, дефисы), карту **H2/H3**.
2. Заложи место под **баннер 21:9** в теле (до или после первого крупного `<h2>` согласно `seo.json`).
3. OG/цитаты для ИИ (короткие извлекаемые абзацы) по желанию ТЗ.

Выход: `seo.json`.

---

## Фаза E — Изображения через MCP (mcp-kv.ru)

Инструменты: **`nano_banana_pro`** или **`nano_banana_2`** (список смотри `npm run mcp:tools-check`). Параметры по умолчанию:

| Роль | aspect_ratio | Примечание |
|------|----------------|------------|
| Обложка | **16:9** | Для поста WordPress как featured после загрузки |
| Баннер в тексте | **21:9** | Вставляется в `<figure class="article-banner">` |

1. Вызови генерацию обложки; из ответа извлеки публичный URL (последний `https://…` или маркер из промпта).
2. Вызови генерацию баннера аналогично.
3. Загрузи файлы в WordPress: предпочтительно **`wordpress_upload_media`** с `file_url` (надёжнее для временных хостов), затем получи **постоянный** URL из медиатеки.
4. Запиши ID медиа обложки для **`featured_media`**.
5. Подставь **постоянный** URL баннера в HTML черновика.

При таймаутах увеличивай **`MCP_REQUEST_TIMEOUT_MS`** (до 900000).

Промпты по смыслу studio/WordPress — см. **`prompts/wordpress-articles/NANO_WORDPRESS_STUDIO.md`**.

Автоматизация после заполнения `artifacts/pipeline-state.json` и создания поста: **`npm run wp:nano-images-republish`**.

## Фаза F — Написание статьи (Writer + Humanizer)

1. Собери текст строго по **`prompts/wordpress-articles/HTML_STRUCTURE_WORDPRAIS.md`** и регламенту человечности (`russian-humanizer-seo`).
2. Соблюдай **ЗАПРЕТЫ** из извлечённого промпта статьи (`prompts/_extracted/012_*`): без эмодзи, без воды, без скрытого промо сторонних школ.
3. Внутренние ссылки и CTA — только **wordprais.ru** (или относительные пути), согласно конфигу.
4. Объём ориентировочно **≥7000 символов** при возможности темы; не раздувай пустыми повторами.

Выход: `article-draft.md` → после humanizer **`article.md`** (если пайплайн Markdown) и параллельно финальный **`articleHtml`** для WordPress в `artifacts/pipeline-state.json` или эквивалент.

## Фаза G — Сборка HTML под WordPress

Перед публикацией проверь:

- [ ] Нет `<h1>`, есть только контент записи.
- [ ] Есть лиды, несколько `<h2>`, минимум одна таблица.
- [ ] Есть `<figure class="article-banner">` с корректным **постоянным** `src`.
- [ ] Два уровня FAQ при необходимости (`Частые вопросы` + опционально второй блок).
- [ ] Блок «Полезные ресурсы» и «Что делать дальше» ссылками только на разрешённые URL.
- [ ] Все изображения с **`alt`**, баннер с **`loading="lazy"`**.

## Фаза H — Антидубль и медиа

1. Прогони **`duplicate-guardian`**: при блокировке измени slug/углы темы.
2. Опционально **`metla-media-cleaner`** если включена политика.

---

## Фаза I — Публикация WordPress (MCP)

1. Убедись что **`artifacts/pipeline-state.json`** содержит `seoTitle`, `metaDescription`, `articleHtml`.
2. Создай или обнови пост: **`wordpress_create_post`** / **`wordpress_update_post`**; передай **`featured_media`** если есть ID загрузки.
3. Убедись что **`wordpress_upload_media`** отработал для баннера и контент ссылается на CDN сайта, а не на tempfile.

Скрипты репозитория:

- `npm run wp:publish-streamable` — потоковая публикация из state.
- `npm run wp:nano-images-republish` — nano + загрузка + обновление поста для уже созданной записи.

---

## Фаза J — После публикации

1. **`content:finalize-publish`** — синхронизация артефактов и IndexNow при настроенном ключе.
2. Supervisor QA: битые ссылки, дубликаты заголовков, скорость загрузки изображений, мобильная читаемость.

---

## Мини-проверка «не сломалось»

- Открыть пост на wordprais.ru: есть featured image, баннер в теле грузится с вашего домена.
- Валидная иерархия заголовков в инструментах разработчика.
- Нет чужих доменов в тексте кроме разрешённых источников фактов.
