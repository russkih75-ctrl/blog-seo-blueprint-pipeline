# duplicate-title-meta-guardian

Используй этот skill как **обязательный отдельный проход** в автоматизации «Вордпресс статьи», когда пользователь или очередь Wordstat требуют гарантию отсутствия дублей.

## Когда включать

- Любая публикация на `wordprais.ru` через `wordpress-articles`.
- После черновика SEO title / meta description / slug и **до** финальной записи в `artifacts/pipeline-state.json` и до `wordpress_create_post`.

## Входные данные

- Предлагаемые значения: `seoTitle`, `metaDescription`, `slug` (или базовый slug из главного ключа).
- `artifacts/content-index.json` — все предыдущие материалы (title, slug, primaryKeyword).
- При доступе к MCP: поиск по сайту через `wordpress_search_posts` по фразам из title и primary keyword.

## Правила решения

1. Нормализуй сравнение: нижний регистр, сжатие пробелов, убрать знаки препинания для сравнения «семантического равенства» заголовков и описаний.
2. **Блокируй публикацию**, если:
   - slug совпадает с любым из индекса (или отличается только морфологией/стоп-словами при том же смысле);
   - seoTitle дубликат или near-duplicate (совпадение ≥ ~92% по токенам или очевидный тот же смысл);
   - meta description совпадает или почти совпадает с любым предыдущим (включая перестановку блоков).
3. При блокировке предложи **конкретные** альтернативы: новый угол в заголовке, уточнение интента (коммерция / обучение / сравнение сценариев), другой slug (`-2026`, `-elementor`, `-podderzhka` и т.д. — без спама и без обмана).
4. Зафиксируй результат в артефактах прогона (например заметка supervisor / qa-report), чтобы следующий этап видел решение.

## Выход

- `pass` — уникально, можно публиковать.
- `rewrite_required` — нужны новые title/meta/slug (передать обратно автору или writer-стадии).

Связанные материалы: `duplicate-guardian` в content-factory (похожесть тела статьи), `prompts/wordpress-articles/MASTER_PROMPT.md` фаза H.
